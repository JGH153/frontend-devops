It's about creating better software faster Go hand in hand with Adgile To get a shorter Systems development life cycle
The problem
Big and infrequant deployments Just fixed a bug just after deploy to prod? Have to waith to next one in a mounth Long time to you know if users like a new feature More changes at the same time can interfeer with eachother
Manual deploys are error prone Forget something everytime How fast can you do it? Under stress?
Long living branches cause merge hell
Production is someone elses problem! Have to talk to a different company to deploy A lack of responsibility, and control
The solution Dev-Ops
Dev-Ops is a process, not a goal Merger of Development and Operations One makes the initial application One gets it to production Merge them togheter, Dev-Ops Mot nessesarily same persion
Dev-Ops is a fluint term, and moving target Many do a few elements and call it devops
The elements of devops ever changing My experience is more on dev than ops side
Continuous Integration (CI) Merge to master often Trunk
Continuous Delivery (CD) Publish to prod on each merge Github Actions
Microservices Many small backend services Microfrontends Still new
Infrastructure as Code Config files and automation of infrastructure Terraform
Monitoring and Logging Know instantly when something went wrong in prod and be able to understand it after it happened.
Communication and Collaboration Have teams talk to each other, and document things.
Continuous feedback Use feedback from production to influence future development
Summary Continuous Integration (CI) Continuous Delivery (CD) Microservices Infrastructure as Code Monitoring and Logging Communication and Collaboration Continuous feedback
